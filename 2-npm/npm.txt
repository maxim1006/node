npm init - создает package.json
npm adduser - создает юзера
npm publish - публикует модуль
npm unpublish - удаляет модуль
npm help - посмотреть команды npm
npm search или npm s имя_модуля - поиск модуля
npm up/npm up имя_модуля - обновляю все модули
npm r имя_модуля - удаляю модуль
npm prune - удаляет лишние модули, которые не перечислены в зависимостях



/*******
Когда я пишу npm i имя_модуля, то нода идет вверх по файловой структуре до места где есть папка node_modules либо package.json и устанавливает модуль туда. Если хочу проинсталлить в папку, то создаю в ней папку node_modules

могу написать npm i express@3.0 - с указанием версии модуля (MAJOR.MINOR.PATCH)
********/



/****
Package.json

в dependencies в package.json указываются зависимости от которых зависит данный модуль, все эти зависимости поставятся в директорию node_modules внутри скаченного модуля.

в devDependencies - не ставятся, если модуль подтягивается как зависимость, а поставятся они если:
- если стоит флаг конфига npm config, если зайти в директорию скаченного модуля (в примере express и скаченный модуль express) и уже в этой директории ввести npm i
*****/


/*******
устанавливаю модуль глобально npm i имя_модуля -g - это значит, что модуль будет установлен в стандартную системную директорию под windows: user/AppData/Roaming/npm

Для работы с глобальными модулями использую теже npm команды, только с флагом -g, например npm -g up
